Project Ideas

OVERVIEW
Imagine having an application to do things for you, where you can initiate the program and have the program detect what kind of software one is working with or have subcategories for their purpose and have a bunch of other tools within the program to then acquire information for the user and filter through what is necessary.
MOTIVATION
The advancement of technology has progressed at godspeed. In 1909, Nikola Tesla predicted that we would have personal wireless devices, in those times, wireless devices seem like science-fiction and in now days, wireless devices have become a norm. According to “The Statistics Portal”, 52.2% of internet website traffic is from the use of these wireless devices. With these advancements, newer technology requires the latest software. To utilise these latest software, more procedures are developed in order to utilise the software and the more the assumed knowledge of an individual in the field of IT requires.
DESCRIPTION
The end product will be a simple application in a small windows box with a comprehensive list of tools that one can use, anywhere from debugging to benchmarking for a newly built computer. There will be a setting within to change the layout to “advanced” for people who have a better grasp on the variety of applications that one can use for their case. Each tool will have tags for them to indicate what the program can be used for such as benchmarking and data shredding. In the ‘basic’ format of the program, the applications will be organised to categories according to the tags, however, more broad. Advanced layout of the program will have the applications in a list in alphabetical order as hyperlinks. When the hyperlinks are clicked, the user will be directed to another page within the application with the tags that the application comes under and with a brief summary of what the application does. When the user wishes to use the application, a download will then commence and when it does, the program will notify the user about the progress about the download and installation process of the application. After downloading and installing the application, the application will open separately to the program and will be run from there on as the application is only meant to serve as a connection between the user and a vast variety of programs that have been approved by people who know that the application will work and perform and do what it is intended.
TOOLS AND TECHNOLOGIES
To create the program to link itself to lots of different applications, there are countless applications online that can be found online. The applications used for the program have to be compiled by either one person with knowledge of that many other applications or a community of people compiling a few each and one or a few people to put the list into the program.
REQUIREMENTS
Knowledge of programming in one or multiple languages for the program Windows Desktop Program can work in unison with the knowledge of programming. There shouldn’t be any special hardware requirement as long as the computer runs with basic requirements such at 4gb of ram, low grade cpu but functions effectively and sufficient memory space. To acquire the skills required to create the program won’t come in a day or two, one needs to have knowledge of programming languages beforehand.
OUTCOME
Creating a program that is compiled through a group of people who are specialised by the IT community will assist the user, beginners and experts, in a way that the user won’t have to search the internet for hours upon hours to finally download the tool that they want/needed. The development of the program will help the users save time and discover more applications that do the same thing as others, better or find an application that they prefer.
